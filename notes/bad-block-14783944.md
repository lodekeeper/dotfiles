# Bad mainnet block — slot 14783944 / EL block 25547425

**Date:** 2026-07-16 19:49:12 UTC
**Observed by:** ChainSafe mainnet node (Lodestar 1.44.0/e3d733b + Erigon)

## TL;DR

An MEV-boost builder block carried a header `gasUsed` **exactly 25,000 higher** than the actual
execution of its own 617 transactions. Every EL rejected it on `engine_newPayload`. The block was
orphaned and the network self-healed in a single slot. **Not a CL or EL client bug** — the bad value
originated at the builder and was delivered by the relay without being caught.

## The defect

```
header gasUsed     = 35,114,738
actual execution   = 35,089,738
difference         =      25,000   (header is higher)
txCount = 617, receiptCount = 617
```

Erigon's per-transaction trace shows `cumGasUsed == computedCumGas` for **every one** of the 617
transactions — no individual transaction diverges. The header total is simply 25,000 above the sum
of the block's actual execution.

  mismatch.

## Verified: not our bug

publicnode independently returns the same canonical `0xcc4ef417…` for block 25547425 — our Erigon
agreed with the rest of the network, so we were never on a minority fork. Head progressed normally.

This is the **only** `execStatus=INVALID` occurrence across our `beacon-2026-07-13..17` logs.

## Unverified hypothesis

25,000 is exactly `G_newaccount` / the EIP-7702 `PER_EMPTY_ACCOUNT_COST` constant. That is a
suggestive coincidence and may point at an empty/dead-account accounting edge case in the builder's
pipeline — **but we have not verified the mechanism.** Treat as a lead, not a conclusion.

Similarly, optimistic relaying would explain why the block wasn't simulated before delivery, but we
have not confirmed ultrasound's builder status for this submission.
